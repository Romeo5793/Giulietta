# Giulietta Service — プロダクト仕様（SPEC）

最終更新: 2026-07-14  
対象: Alfa Romeo Giulietta 1.4 コンペティツィオーネ MT（MultiAir）  
対象読者: 自分 / Cursor（実装時は本ファイルを正とする）

---

## 1. ひとことで言うと

**自分の Giulietta の整備タイミングを逃さない、Webの車検証・整備手帳。**

OBD（安いBLEアダプタ）で今の状態も見られる。

---

## 2. コア感情

ユーザーが最初の約30秒で感じるべきこと:

> 「次に何をすればいいか、一目で分かる」

判定:

- その機能は「次の整備・いまの車の状態」を分かりやすくするか？ → **残す**
- 診断チューナー／SNSポータルに見えるか？ → **切る / 後回し**

---

## 3. 想定ユーザー

- この車を自分で世話するオーナー（まず1台）
- 技術に詳しくなくても使える（専門用語は二次情報）
- Web（Chrome / Edge）で使う。Mac 取得後にアプリ化を検討

---

## 4. やる / やらない

### やる（いまの幹）

- 車両プロフィール（走行距離・年式メモ）
- 1.4 MultiAir 向けの定期整備メニューとリマインド（距離 or 月数）
- 整備実施記録（日付・距離・費用・メモ）
- Web Bluetooth による安価な **BLE** ELM327 系アダプタ接続
- ライブ簡易メーター（回転・速度・冷却水・ブースト系など）
- DTC 読み取り（と消去は確認付き）
- 任意で Gemini に「このコード／症状どう見る？」と聞く（キーはユーザー入力）
- データはブラウザ `localStorage`（クラウドなし）

### やってもよい（後続）

- CSV / 印刷向け整備履歴エクスポート
- 写真メモ
- Flutter などでのスマホアプリ化（Mac 環境後）
- Web Serial（USB）接続の追加

### やらない

- 工場出荷レベルの診断ツール代替の主張
- クラシックBluetooth（SPP）のみのアダプタへのブラウザ直結（API制約）
- 猫翻訳など別ジャンル機能の混在
- 過大な「故障を完璧に特定」表現

## 6. 画面構成

1. **ホーム** — 次に来る整備・現在距離の更新
2. **整備** — メニュー一覧・記録追加・履歴
3. **OBD** — 接続 / デモ / 簡易メーター / DTC
4. **設定** — 車両情報・Gemini APIキー・データ消去

---

## 7. 技術方針

### いま

- 静的 Web（`index.html` / `css` / `js`）
- HTTPS または localhost で Web Bluetooth が動く環境（Chrome / Edge 推奨）
- アダプタは **BLE** 対応の ELM327 系（FFF0 / FFE0 など）を想定
- iPhone Safari は Web Bluetooth 非対応が多い → ブラウザ案内を表示

### あとで

- Mac 上でアプリ化（Flutter 等）を検討
- 個人APIキー入力に依存しない構成は公開時に再検討

---

## 8. 成功指標（初期）

- 次の整備項目がホームで分かる
- 整備を1件記録できる
- BLEアダプタ（またはデモ）でメーター／DTCが見える
